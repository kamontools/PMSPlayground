InputText

Text input component for user data entry

Overview

Purpose

InputText is a single-line text input component used for collecting user-entered text data. It supports validation states, disabled state, and provides clear visual feedback for user interactions.

When to Use

  • Collecting short text input (names, codes, numbers)
  • Form fields requiring text entry
  • Search inputs (when combined with search icon)
  • Any scenario requiring single-line text input

When Not to Use

  • Multi-line text input (use Textarea component instead)
  • Selecting from predefined options (use Dropdown instead)
  • Date/time input (use Calendar component instead)

Variants

Default

Standard enabled state with white background and grey border.

Disabled

Non-interactive state with light grey background and muted text.

Error

Validation error state with red border and error message.

Error message text

Focused

Active focus state with primary blue border.

Usage Guidelines

For Designers

Visual Design
  • Always include a clear label above the input field
  • Use placeholder text sparingly and only for helpful hints
  • Maintain consistent spacing: 4px gap between label and input
  • Use required indicator (*) in error color (#F03737) for mandatory fields
Layout Considerations
  • Input height is fixed at 40px for consistency
  • Maintain 8px vertical spacing between form fields
  • Error messages appear 4px below the input field
  • Align labels consistently (left-aligned is standard)
Content Guidelines
  • Labels should be concise and descriptive
  • Error messages should be specific and actionable
  • Placeholder text should not replace labels
  • Use helper text (12px, #585A5C) for additional context when needed

For Developers

Implementation
  • Use PrimeVue InputText component as base
  • Apply Pass Through API for custom styling
  • Implement validation logic for error states
  • Ensure proper form integration with v-model
Props API
Prop Type Default Description
modelValue String '' Two-way binding value
label String undefined Label text displayed above input
required Boolean false Shows required indicator (*)
disabled Boolean false Disables the input field
error Boolean false Shows error state styling
errorMessage String '' Error message text displayed below input
placeholder String '' Placeholder text
Events
Event Payload Description
update:modelValue value: String Emitted when input value changes
blur event: FocusEvent Emitted when input loses focus
focus event: FocusEvent Emitted when input gains focus

Accessibility Considerations

Keyboard Interaction

Key Action
Tab Moves focus to the input field
Shift + Tab Moves focus to previous element
Arrow Keys Navigate within text (when focused)
Home / End Move cursor to start/end of text
Escape Clear input (if implemented)

Screen Reader Behavior

  • Role: textbox (implicit from input type="text")
  • Label: Associated label is announced when field receives focus
  • State: Disabled state is announced; error state should be communicated via aria-invalid and aria-describedby
  • Required: aria-required="true" should be set for required fields
  • Error Message: Error message should be linked via aria-describedby

Contrast and Focus Rules

  • Focus Indicator: Primary blue border (#1C70F7) with 1px width provides clear focus indication
  • Contrast Ratio: Text color (#181819) on white background meets WCAG AA (21:1)
  • Placeholder Contrast: Placeholder text (#B4B4B4) meets WCAG AA minimum (4.5:1)
  • Error Text: Error message color (#F03737) meets WCAG AA contrast requirements
  • Focus Order: Input should follow logical tab order within form

Spacing and Layout

Padding

  • Internal Padding: 8px vertical, 12px horizontal (total height: 40px)
  • Label Padding: No padding, positioned 4px above input
  • Error Message Padding: No padding, positioned 4px below input

Margins

  • Default Margin: 8px vertical spacing between form fields
  • Spacing from Adjacent Elements: 8px minimum gap
  • Container Margin: Follows parent container spacing rules

Alignment

  • Text Alignment: Left-aligned text within input
  • Label Alignment: Left-aligned above input
  • Error Message Alignment: Left-aligned below input
  • Responsive Behavior: Full width on mobile, constrained by container on desktop

Dimensions

  • Minimum Width: 200px recommended
  • Maximum Width: Container width or 100%
  • Height: Fixed at 40px
  • Responsive Sizing: Width adapts to container, height remains constant

Usage Examples

Common Pattern 1: Basic Text Input

Scenario: Simple form field for user name entry

Vue.js Implementation 
<template>
  <div class="input-wrapper">
    <label v-if="label" class="input-label">
      {{ label }}
    </label>
    <InputText
      :model-value="modelValue"
      @update:model-value="$emit('update:modelValue', $event)"
      :placeholder="placeholder"
      :pt="{
        root: { class: 'custom-input' }
      }"
    />
  </div>
</template>

<script setup>
import InputText from 'primevue/inputtext'

defineProps({
  modelValue: String,
  label: String,
  placeholder: String
})

defineEmits(['update:modelValue'])
</script>

Common Pattern 2: Required Field with Validation

Scenario: Form field with required indicator and error handling

Vue.js Implementation 
<template>
  <div class="input-wrapper">
    <label v-if="label" class="input-label">
      {{ label }}
      <span v-if="required" class="required-indicator">*</span>
    </label>
    <InputText
      :model-value="modelValue"
      @update:model-value="handleUpdate"
      @blur="validate"
      :class="{ 'p-inputtext-error': error }"
      :pt="{
        root: { class: 'custom-input' }
      }"
    />
    <span v-if="error && errorMessage" class="error-message">
      {{ errorMessage }}
    </span>
  </div>
</template>

<script setup>
import { ref } from 'vue'
import InputText from 'primevue/inputtext'

const props = defineProps({
  modelValue: String,
  label: String,
  required: Boolean,
  errorMessage: String
})

const emit = defineEmits(['update:modelValue'])

const error = ref(false)

const handleUpdate = (value) => {
  emit('update:modelValue', value)
  if (error.value) {
    validate()
  }
}

const validate = () => {
  if (props.required && !props.modelValue) {
    error.value = true
  } else {
    error.value = false
  }
}
</script>

<style scoped>
.required-indicator {
  color: #F03737;
}

.p-inputtext-error {
  border-color: #F03737;
}

.error-message {
  font-family: 'Sarabun', sans-serif;
  font-size: 12px;
  line-height: 20px;
  color: #F03737;
}
</style>

Edge Case: Disabled State

Scenario: Input field that should be non-editable based on user permissions or workflow state

Vue.js Implementation 
<template>
  <div class="input-wrapper">
    <label class="input-label">{{ label }}</label>
    <InputText
      :model-value="modelValue"
      :disabled="disabled"
      :pt="{
        root: { 
          class: 'custom-input',
          style: disabled ? 'background: #F2F6FC; color: #84868A;' : ''
        }
      }"
    />
  </div>
</template>

<script setup>
import InputText from 'primevue/inputtext'

defineProps({
  modelValue: String,
  label: String,
  disabled: Boolean
})
</script>

Considerations:

  • Disabled inputs should still be visible to show context
  • Consider showing a tooltip explaining why the field is disabled
  • Ensure disabled state is communicated to screen readers

With Label and Validation

Vue.js 
<template>
  <div class="form-field">
    <label class="field-label">
      Username
      <span class="required">*</span>
    </label>
    <InputText
      v-model="username"
      :class="{ 'error': hasError }"
      @blur="validate"
    />
    <span v-if="hasError" class="error-text">
      Username is required
    </span>
  </div>
</template>

<script setup>
import { ref } from 'vue'
import InputText from 'primevue/inputtext'

const username = ref('')
const hasError = ref(false)

const validate = () => {
  hasError.value = !username.value
}
</script>
